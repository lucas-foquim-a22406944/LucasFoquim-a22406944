Sobre esta Aplicação

1. Arquitetura MVT do Django

Model

O Model representa a camada de dados da aplicação. Define a estrutura da base de dados através de classes Python, e é responsável por guardar, consultar e manipular os dados.

View

A View é a camada lógica. Recebe os pedidos HTTP do browser, consulta os dados necessários ao Model, e envia-os para o Template para serem renderizados.

Template

O Template é a camada de apresentação. É um ficheiro HTML com tags especiais do Django que permitem inserir dados dinâmicos vindos da View.

Fluxo de informação

O browser faz um pedido (request) → o Django procura a URL em urls.py → chama a View correspondente → a View consulta o Model → os dados são enviados para o Template → o Template renderiza o HTML → o browser recebe a resposta (response).

2. Modelação

Diagrama Entidade-Relação

models.py

class Tecnologia(models.Model):
    nome = models.CharField(max_length=100)
    tipo = models.ForeignKey(TipoTecnologia, ...)
    descricao = models.TextField()
    nivel_interesse = models.IntegerField()

class Competencia(models.Model):
    nome = models.CharField(max_length=200)
    tipo = models.CharField(max_length=50)
    nivel = models.CharField(max_length=50)
    tecnologias = models.ManyToManyField(Tecnologia)

class Projeto(models.Model):
    titulo = models.CharField(max_length=200)
    descricao = models.TextField()
    ano = models.IntegerField()
    tecnologias = models.ManyToManyField(Tecnologia)
    competencias = models.ManyToManyField(Competencia)

class Formacao(models.Model):
    titulo = models.CharField(max_length=200)
    instituicao = models.CharField(max_length=200)
    data_inicio = models.DateField()
    competencias = models.ManyToManyField(Competencia)
